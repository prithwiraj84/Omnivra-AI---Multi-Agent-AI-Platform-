# workspace/ — AI Agent Output ONLY

> **WORKSPACE RULE.** AI agents may write artifacts **only** under this directory. They must
> **NEVER** modify project source code (`/frontend`, `/backend`, `/supabase`, root configs).

## Subdirectories
| Path | Contents |
|---|---|
| `workspace/frontend/` | Draft frontend artifacts generated by agents (reviewed before promotion). |
| `workspace/backend/` | Draft backend artifacts generated by agents. |
| `workspace/docs/` | Agent-authored documentation drafts. |
| `workspace/presentations/` | Generated decks / exports (pre-approval). |
| `workspace/reports/` | Generated reports (SEO, QA, security, etc.). |
| `workspace/.state/` | **Runtime** state mirror: live `project_state.json`, `checkpoints/`, `artifacts/`. Volatile + git-ignored. |

## Notes
- The backend filesystem service path-jails all agent writes to `WORKSPACE_DIR` (this folder).
  Any attempt to write outside it is rejected.
- Everything here except `README.md` / `.gitkeep` is git-ignored. The **durable** project state
  lives in `/docs` (`PROJECT_STATE.md`, `FILE_MANIFEST.md`, `CHECKPOINTS.md`, `ROADMAP.md`).
- Promotion from `workspace/` to a source tree happens **only** via a human Approval Gate decision.
